Cider

Getting Started

Cider is a combination of homebrew and chef to fully manage your local development environment on your mac. It's opinionated in a few ways but don't worry, it installs to its own chroot'd environment so you can try it without committing your current development environment to it. If you already have rvm installed you're halfway there.

If you have rvm installed simply install cider.

% gem install cider --no-rdoc --no-ri

If you don't have rvm installed, install with sudo.

% sudo /usr/bin/gem install cider --no-rdoc --no-ri

Run the cider executable.

% cider

If everything executes properly you'll see messages coming from chef during the cider run. It will take a while on the first run but it's really quick after that. Right now the recipes are hard coded but I hope to add support to allow developers to build their own flavors for their machines. Ideally everyone just picks and what they want on their workstations and excludes the stuff they don't want.

This might not be what you need right now, but if you have to bootstrap a new environment I suggest giving it a try. The entire environment should be able to be thrown away and recreated with no more than a single shell call to the cider executable. Think about it. :)

What It Gives You

  • mysql running as your user
  • postgres daemon running as your user
  • mongodb running as your user
  • rvm with rubinius and latest stable ruby.
  • node.js development environment

Notes

  • Backups should be made for any dotfiles cider touches.
  • Customizing PS1/Shell Env/etc: 
    cinderella% cat ~/.cider.profile.custom
set -o vi
if [[ -x ~/Developer/bin/fortune ]]; then
  echo ""; fortune; echo ""
fi
alias nerd="vim +NERDTree"
alias g="git"
  • Sepcifying the default RVM Environment:
    cinderella% DEFAULT_RVM_VM=ruby-1.9.2 cider
