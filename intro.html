Cider

Getting Started

Cider is a fully managed development environment for open source hacking on Mac OSX. It's powered by rvm, homebrew and chef. You only need Xcode to get started.

Cider builds everything up in ~/Developer. It won't stomp on any of your current installations so you don't have to commit your entire machine to cider immediately. It's a breeze to rollback if you really want to.

Bootstrap yourself from curl:

% curl \
   http://github.com/atmos/cider/raw/master/bootstrap.sh \
   -o - | sh

It will take about an hour to run the first time. Subsequent runs take fewer than 15 seconds. When it completes double check the following.

Check your ruby version:

% ruby --version

It should resemble something like this:

ruby 1.8.7 (2010-01-10 patchlevel 249) [i686-darwin10.3.0]

Ensure the new cider is installed:

% which cider

It should resemble something like this:

/Users/atmos/.rvm/gems/ruby-1.8.7-p249/bin/cider

Ruby 1.8.7 from rvm is now your default version of ruby. Cider should be able to manage itself from here. You're ready to go hack and do stuff. Checkout github perhaps.

Running Cider

After the initial bootstrapping you only need to run the cider executable. You should run cider regularly to keep up with the latest changes. Cider will automatically update itself if when needed. Currently there's no command line options, it just installs and exits. 

% cider
INFO: Starting Chef Solo Run
INFO: Replacing the run_list with ["homebrew", "rvm", "node"] from JSON

I hope to add support to allow developers to build their own flavors for their machines. Ideally everyone just picks how they want their development environment configured.

This might not be what you need right now, but if you have to bootstrap a new environment I suggest giving it a try. The entire environment should be able to be thrown away and recreated with no more than a single shell call to the cider executable.
