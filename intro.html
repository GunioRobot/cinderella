Cider

Getting Started

Cider is a combination of homebrew and chef to fully manage your local development environment on your mac. It's opinionated in a few ways but don't worry, it installs to its own chroot'd environment so you can try it without committing to ruining your current development environment. 

          % sudo gem install cider --no-rdoc --no-ri
          % cider

If everything executes properly you'll see messages coming from chef during the cider run. It will take a while the first run but it's really quick after that. Right now the recipes are hard coded but I hope to add support to allow developers to build their own runlists for their machines.

This might not be what you need right now, but if you have to bootstrap a new environment I suggest giving it a try. Pretty much the entire environment should be able to be thrown away and recreated with no more than a single shell call to the cider executable. Think about it. :)

What It Gives You

  • mysql running as your user
  • postgres daemon running as your user
  • mongodb running as your user
  • rvm with rubinius and latest stable ruby.
  • node.js development environment

Notes

  • Customizing PS1/Shell Env/etc: use ~/.cider.profile.custom, it won't be overwritten each run.
  • Sepcifying the default RVM Environment: DEFAULT_RVM_VM=ruby-1.9.2 cider
  • Backups should be made for any files cider touches.
