Cinderella

What It Gives You

  • mysql, postgres, redis, memcached and mongodb
  • ruby (1.8.7) with rails and sinatra via rvm gemsets.
  • python (2.7) with easy_install.
  • node.js (0.2.0) with npm.
  • erlang (R13B04) environment.

Getting Started

Cinderella is a fully managed development environment for open source hacking on Mac OSX. It's powered by rvm, homebrew and chef. You only need Xcode to get started.

Cinderella builds everything up in ~/Developer. It won't stomp on any of your current installations so you don't have to commit your entire machine immediately. It's simple to rollback if you really want to.

Bootstrap with curl

Just curl this URL. If the bootstrap script needs any interaction with you with you it will be in the first few seconds. It might need sudo to install the cinderella executable.

The build should take about 45 minutes. 

% curl \
   http://github.com/atmos/cinderella/raw/master/bootstrap.sh \
   -o - | sh
[22:06:02] INFO: Starting Chef Solo Run
[22:34:29] INFO: Chef Run complete in 1707.521717 seconds
%

Cinderella can manage itself from here. Open up a new shell and you're ready to hack.

Running Cinderella

Just run it, it updates itself. 

% cinderella
INFO: Starting Chef Solo Run
INFO: Replacing the run_list with ["homebrew", "rvm", "node"] from JSON

Ideally everyone just picks and chooses how they want their development environment configured.

Notes

  • Backups are created for any dotfiles cinderella touches.
  • Customize your environment in ~/.cinderella.profile.custom
  • Sepcifying a default RVM version other than 1.8.7: 
    % rvm use 1.9.2 --default
  • Cinderella used to be called cider, a very serious software company felt that I was infringing upon their trademark so I've now switched the name to Cinderella.
